# Security Policy

## Supported Scope

This repository contains the **TaktX Community Console** and its related frontend, backend, and Docker deployment assets.

## Reporting a Vulnerability

Please **do not** disclose security vulnerabilities in public GitHub issues, pull requests, or discussions.

Instead:

1. Use a private vulnerability reporting feature on the repository host if one is available.
2. If no private reporting channel is available, open a minimal public issue **without technical details** and request a secure contact path from the maintainers.
3. Include enough information for maintainers to reproduce and assess the issue once a private channel is established.

Recommended information to include in a private report:

- Affected component (`frontend`, `platform-service`, `ingester`, `docker`, etc.)
- Impact and severity estimate
- Reproduction steps or proof of concept
- Version, branch, image tag, or commit SHA affected
- Suggested remediation, if known

## Response Expectations

Maintainers will try to:

- Acknowledge reports in a reasonable timeframe
- Reproduce and assess the issue
- Coordinate a fix and disclosure approach where appropriate

Response times may vary because this is a community-maintained open-source project.

## Disclosure

Please allow maintainers reasonable time to investigate and remediate before public disclosure.
